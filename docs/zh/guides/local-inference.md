# 本地推理 0 Tokens 方案

当智能体采用**本机（local）**执行并选用**本地 GGUF 模型**时，推理完全在本机由 [node-llama-cpp](https://github.com/nicolo-ribani/node-llama-cpp) 完成，**不调用任何云端 API**，因此 **0 Token 消耗**。本文说明配置、使用方式，并重点介绍 **一键省 Token** 特征。

---

## 一键省 Token 特征（重点）

- **含义**：在设置中将默认（或某智能体）的 **Provider 选为 local、模型选为已下载的本地 GGUF**，即可实现「对话不花任何云端 Token」；无需改代码、无需代理到第三方，本机即可完成问答、工具调用与技能执行。
- **适用场景**：
  - 私密或敏感对话，不希望请求发到云端
  - 云端 API 配额有限或按量计费，希望日常对话零成本
  - 离线或内网环境，无法访问云端 API
- **与代理 0 Token 的区别**：代理模式（Coze/OpenCode 等）是「本机 0 消耗、推理在对方平台」；本地推理是「推理也在本机、全程 0 云端 Token」。

---

## 配置步骤

### 1. 下载并管理本地模型

1. 打开 **设置 → 模型配置 → 本地模型**。
2. 在「推荐模型」中点击 **中国** 或 **全球** 下载所需模型，等待下载完成；文件保存在 **`~/.openbot/.cached_models/`** 目录。
3. 在「已安装的本地模型」中可查看已下载文件；展示名称与推荐列表一致。
4. 若缺省模型文件不存在，Gateway 启动时不会自动启动本地服务；需在本页先下载或确认已有模型后，再执行下一步。

### 2. 启动本地模型服务

1. 在同一页「启动本地模型服务」区域：
   - 查看状态：**本地模型服务已就绪** 表示已在运行；**不可用** 表示未启动。
   - 在 **LLM 模型**、**Embedding 模型** 下拉中可选「使用默认」或指定已安装的某一模型；**至少选择其一**即可启动（仅 LLM、仅 Embedding 或两者都选均可，有一个就启动一个）。
   - 点击 **启动本地模型服务**（若已就绪则显示「重启本地模型服务」），等待就绪。
2. 服务就绪后，已加载的 LLM 提供对话、已加载的 Embedding 提供向量，不占用云端 Token。

### 3. 将智能体设为使用本地模型

1. 打开 **设置 → 智能体**，编辑要使用本地推理的智能体（或默认主智能体）。
2. **执行方式** 选择 **本机（local）**。
3. 打开 **设置 → 模型配置**，在「模型配置」中为 **local** Provider 配置模型：
   - 若尚未配置，可新增一条：Provider 选 **local**，模型从下拉中选已安装的本地模型（名称与「已安装的本地模型」一致）。
   - 将该 local 模型设为默认，或在智能体编辑页的「模型」中选择该 local 模型。
4. 可选：在智能体编辑页或模型配置中设置 **上下文长度**（默认 32K），仅对本机 local 生效。

完成以上步骤后，该智能体的对话与工具调用均在本机执行，**0 云端 Token 消耗**。

---

## 使用方式

- **桌面端 / Web 端**：将默认智能体或当前会话智能体设为上述「local + 本地模型」智能体，正常发起对话即可；回复由本机 GGUF 生成。
- **通道（飞书/钉钉/Telegram/微信）**：在通道配置中将该通道的默认智能体设为该 local 智能体，则该通道对话也走本地推理，0 Token。
- **CLI**：在 `~/.openbot/desktop/` 配置中 defaultAgent 或对应 agent 的 provider 为 local、model 为本地模型文件名（或 modelItemCode 对应 local 的已配置模型），执行 `openbot chat` 等即可。

---

## 常见问题

- **本地服务未就绪 / 不可用**：检查是否已在「本地模型」页点击「启动本地模型服务」；若缺省模型文件不存在，先下载或选择已安装模型后再启动。
- **首包慢或超时**：本地模型冷启或首 token 较慢时，可适当延长前端超时（当前为 3 分钟）；流式启动后以 agent_end 判断结束，首包超时单独为 90 秒。
- **推荐列表与已安装名称一致**：推荐模型来自 `presets/recommended-local-models.json`，已安装列表会匹配并显示与推荐一致的名称；已安装的不会出现在「推荐下载」列表中。
